The rule was never the hard part.

Anyone can write one threshold. The value is everything that one line ignores:

The AI can't bypass it

The decision runs in deterministic code outside the model. A prompt injection can fool the agent — it can't fool the gate or talk its way past it.

It catches the real attacks

Drains via many small "plausible" payments, money to never-authorized payees, payloads smuggled into fields. A single amount check is blind to all of them.

Authorization, not just a number

Money moves only under a mandate the human created in advance — scoped to a payee and purpose, capped, expiring, and revocable any time.

Tamper-evident proof

Every decision — allowed or blocked — is recorded in a hash-chained audit trail. You can prove what happened, and detect if anyone edited the log.

Like bcrypt for passwords: simple in spirit, easy to get dangerously wrong, so you use the tested one instead of rolling your own.

Use it in one line

MCP server — Claude Code, Cursor, Claude Desktop, Windsurf…npx -y payment-guard
Library — call it from your own backendnpm install payment-guard

Non-custodial: PaymentGuard never holds or moves your money. It's the decision + audit layer you put in front of whatever rail settles — cards, bank, or stablecoins.
